0.1+0.2的结果为什么是17位小数

我们知道0.1+0.2 == 0.30000000000000004，为什么偏偏是17位，并且为什么偏偏就它是17位呢？

16位安全范围

JS里数值有安全范围，是2^53-1至-2^53+1。2^53-1是9007199254740991，'9007199254740992'.length == 16。

并且，除了0.1+0.2，其它大多数都是16位小数。

0.7+0.1 == 0.7999999999999999 // 16位
1/3 == 0.3333333333333333 // 16位
